# 히스토리 기능 추가 계획

## 확정된 요구사항

| 항목 | 결정 |
|------|------|
| 표시 위치 | Result 화면 하단 (문제 그리드 아래) |
| 기록 항목 | 날짜/시간, 점수(X/50), 남은 시간 |
| 보관 개수 | 최근 10건 |
| Best Score | 최고 점수 행을 별도 강조 (금색) |
| 저장 방식 | `localStorage` (서버 불필요) |

---

## UI 레이아웃 (Result 화면)

```
┌─────────────────────────────────────────────┐
│  ⏱ 1:42    🎯 47 / 50    [ Reset ]          │  ← 헤더
├─────────────────────────────────────────────┤
│  [ 문제 그리드 - 채점 결과 ]                 │
├─────────────────────────────────────────────┤
│  ── 기록 ──────────────────────────────────  │
│  ★  2026-06-19  21:30   50/50   남은 2:05  │  ← Best (금색)
│     2026-06-19  21:28   47/50   남은 1:42  │  ← 이번 결과 (굵게)
│     2026-06-18  19:10   32/50   타임아웃   │
│     ...                                     │
└─────────────────────────────────────────────┘
```

---

## 데이터 구조

```js
// localStorage key: 'madMinuteHistory'
// 저장 형식: JSON array (최신순, 최대 10건)
[
  {
    date: '2026-06-19T21:30:00',  // ISO string
    score: 47,                    // 맞은 개수
    timeLeft: 102,                // 제출 시 남은 초 (0이면 타임아웃)
  },
  ...
]
```

## 구현 변경 사항

### `app.js` 추가 함수

| 함수 | 역할 |
|------|------|
| `saveHistory(score, timeLeft)` | 현재 결과를 localStorage에 저장, 10건 초과 시 오래된 것 삭제 |
| `loadHistory()` | localStorage에서 기록 읽기 |
| `renderHistory()` | Result 화면에 히스토리 테이블 렌더링 |

### `submitAnswers()` 수정

- 채점 완료 후 `saveHistory(score, state.timeLeft)` 호출
- `renderHistory()` 호출

### `index.html` 추가

- `#history-section` 영역 추가 (Result 화면 내 game-screen 하단)

### `style.css` 추가

- `#history-section` 스타일
- `.history-best` — 금색 강조
- `.history-current` — 현재 결과 굵게
